Размер экранного шрифта
        

При наведении курсора на объект, выделенный синим цветом,
показывается назваие тэга этого объекта и его свойства,
имеющие отношение к размеру: заданные
и рассчитанные браузером (в скобках).

  • На мобильных устройствах с высокой плотностью пикселей (Retina-дисплеи) физическое разрешение может быть в 2–3 раза больше логического. Это нужно учитывать при работе с screen.width и screen.height.
  • Размеры окна браузера и физические размеры экрана могут существенно отличаться, особенно на настольных компьютерах, где пользователи редко разворачивают браузер на весь экран.
  • На разных операционных системах свойство screen.availWidth/availHeight может вести себя по-разному.

Размер шрифта

На этой странице применены 4 единицы размера шрифта:

  1. px — пиксели

    Тип: абсолютная

    Поддержка: везде

    Применение: повсеместно 

    body {
  font-size: 16px;
  margin: 20px;
}

    Когда использовать:

    Пиксели подходят для точных размеров, таких как иконки, кнопки и UI-элементы. Часто применяются как базовая единица, особенно на макетах.

    Когда избегать:

    Не стоит использовать для шрифтов — это мешает доступности. Также нежелательны в адаптивной вёрстке: не масштабируются под размер экрана.

    Совет:

    Используйте умеренно, для точных деталей.

  2. % — проценты

    Тип: относительная

    Применение: ширина, отступы, шрифты (редко) 

    .container {
  width: 80%;
}

    Когда использовать:

    Идеальны для ширины элементов, особенно вложенных блоков, которым нужно наследовать размеры от родителя.

    Когда избегать:

    В высоте могут вести себя непредсказуемо, если родитель не имеет заданной высоты.

    Совет:

    Отлично работает для ширины, особенно в гибких макетах.

  3. rem — относительная к корневому шрифту

    Представляет собой размер шрифта (font-size) корневого элемента (как правило, элемента <html>). Если используется в свойстве font-size корневого элемента, то представляет его начальное значение.
    По умолчанию в браузерах используется значение 16px, но пользовательские настройки могут его изменить.

    Тип: относительная

    Применение: шрифты, отступы, размеры, ширина контейнеров 

    html {
  font-size: 16px;
}

h1 {
  font-size: 2rem; /* 32px */
}

    Когда использовать:

    Хорош для масштабируемых интерфейсов: одно изменение font-size в html масштабирует всё остальное. Размер базового шрифта нужно задавать не в body, а в html. Только отсюда берётся размер для рассчёта.

    Когда избегать:

    Почти не бывает причин избегать — универсален и удобен.

    Совет:

    Рекомендуется как основная единица для всех размеров.

  4. vw / vh — проценты от вьюпорта

    Тип: относительная

    • 1vw = 1% от ширины окна
    • 1vh = 1% от высоты окна
    .hero {
  height: 100vh;
  font-size: 5vw;
}
    Когда использовать:
    Отличны для полноэкранных секций, динамичных заголовков и блоков, завязанных на размер экрана.
    Когда избегать:
    На мобильных могут вести себя нестабильно — адресная строка влияет на размер окна.
    Совет:
    Используйте с осторожностью, особенно vh на iOS — лучше lvh или JS-хаки.

Разница между 100% и 100vh/100vw в том, что 100% устанавливается относительно родительского элемента, внутри которого происходит вычисление длины и ширины для данного объекта, в то время как значение 100vh/100vw — абсолютное для окна браузера. 100vh обозначает всю высоту окна браузера, а 100vw — всю ширину. Это различие важно.

 

Ширина объекта  

¯\_(ツ)_/¯

Задать ширину объекта в зависимости от ширины экрана можно с помощью медиазапросов.

  • Прописать стиль для разных размеров окна. Например, если ширина экрана меньше 768px, можно использовать другой стиль: 
    @media (max-width: 768px) {
.myImage {
width: 100% 
}
}

    Этот блок CSS будет применяться только если ширина экрана меньше 768 пикселей.


  • Применить стили только для определённого диапазона ширины. Например, для экранов от 768px до 991px, можно использовать медиа-запрос: 
    @media (min-width: 768px) and (max-width: 991px) {

/* стили для экранов шириной от 768px до 991px */

}

  • Применить стили для определённой ориентации экрана – вертикальной или горизонтальной: 
    @media screen and (orientation: portrait) {
article {
	width: 94vw;
}
}
@media screen and (orientation: landscape) {
article {
	width: 739px;
}
}

  • Стили для устройств с высоким DPI (Retina-дисплеи): 
    @media screen and (min-resolution: 2dppx) {
.logo {
background-image: url('logo@2x.png');
}
}

  • Сочетание стилей на примере стилей для планшетов альбомной ориентации: 
    @media screen and (min-width: 768px) and (max-width: 1024px) and (orientation: landscape) {
.sidebar {
width: 30%;
}
.content {
width: 70%;
}
}

  • Существует множество экранов и устройств с разной высотой и шириной, поэтому трудно создать точную точку разрыва для каждого устройства. Чтобы упростить задачу, вы можете ориентироваться на пять групп: 

    /* Очень маленькие устройства (телефоны 600px и менее) */
@media only screen and (max-width: 600px) {...}

/* Маленькие устройства (вертикально ориентированные планшеты и большие телефоны 600px и больше) */
@media only screen and (min-width: 600px) {...}

/* Средние устройства (горизонтально ориентированные планшеты 768px и больше) */
@media only screen and (min-width: 768px) {...}

/* Крупные устройства (ноутбуки и стационарные компьютеры 992px и болше) */
@media only screen and (min-width: 992px) {...}

/* Очень крупные устройства (крупные ноутбуки и стационарные компьютеры 1200px и больше) */
@media only screen and (min-width: 1200px) {...}

  • Список через запятую: добавьте дополнительный медиа-запрос к уже существующему, используя запятую (это будет работать как оператор ИЛИ):

    Когда ширина находится между 600px и 900px ИЛИ выше 1100px — изменить внешний вид <div> 
    @media screen and (max-width: 900px) and (min-width: 600px), (min-width: 1100px) {
  div.example {
    font-size: 50px;
    padding: 50px;
    border: 8px solid black;
    background: yellow;
  }
}
Медиазапросы могут не работать на мобильных устройствах, если отсутствует или некорректен мета-тег viewport. Решение: добавить в <head> страницы: 
<meta name="viewport" content="width=device-width, initial-scale=1">
